================================================================================
  GUÍA DE INSTALACIÓN EN PC NUEVA - WPE Dashboard
================================================================================

Esta guía te ayudará a instalar el dashboard en una PC nueva donde nunca se ha
ejecutado antes.

================================================================================
  MÉTODO 1: INSTALACIÓN AUTOMÁTICA (RECOMENDADO)
================================================================================

1. Copia la carpeta completa "WPE-Dashboard" a la PC nueva
   - Puede ser desde un USB
   - O desde una carpeta de red: \\servidor\WPE-Dashboard

2. Haz doble clic en:
   → Instalar-Dependencias.bat

3. Windows te pedirá permisos de administrador:
   → Clic en "Sí" para continuar

4. El instalador hará lo siguiente automáticamente:
   ✓ Verificar permisos administrativos
   ✓ Verificar versión de PowerShell
   ✓ Instalar UniversalDashboard.Community v2.9.0
   ✓ Crear carpetas necesarias (Logs, Backup, Temp)
   ✓ Verificar que todo funcione correctamente

5. Espera a que termine (puede tardar 2-5 minutos)
   → Verás mensajes en verde cuando todo esté OK

6. Al finalizar, presiona cualquier tecla para cerrar

7. ¡Listo! Ahora puedes iniciar el dashboard:
   → Doble clic en: Iniciar-Dashboard.bat

================================================================================
  MÉTODO 2: INSTALACIÓN MANUAL
================================================================================

Si prefieres hacerlo manualmente o el instalador automático falla:

PASO 1: Abrir PowerShell como Administrador
   → Haz clic derecho en el botón de Windows
   → Selecciona "Windows PowerShell (Administrador)"
   → Acepta el control de cuentas de usuario

PASO 2: Instalar el módulo UniversalDashboard
   Ejecuta este comando:

   Install-Module -Name UniversalDashboard.Community -RequiredVersion 2.9.0 -Force

   → Si pregunta por PSGallery, responde "Y" (Yes)
   → Espera a que termine la descarga e instalación

PASO 3: Verificar la instalación
   Ejecuta estos comandos:

   Import-Module UniversalDashboard.Community
   Get-Module UniversalDashboard.Community

   → Si ves información del módulo, la instalación fue exitosa
   → Si ves un error, repite el PASO 2

PASO 4: Navegar a la carpeta del dashboard
   cd C:\WPE-Dashboard

PASO 5: Iniciar el dashboard
   .\Iniciar-Dashboard.bat

================================================================================
  SOLUCIÓN DE PROBLEMAS COMUNES
================================================================================

PROBLEMA: "El módulo no se encuentra"
SOLUCIÓN:
   1. Abre PowerShell como administrador
   2. Ejecuta: Install-Module -Name UniversalDashboard.Community -Force
   3. Espera a que termine
   4. Vuelve a intentar

PROBLEMA: "Requiere PowerShell 5.1 o superior"
SOLUCIÓN:
   1. Verifica tu versión: $PSVersionTable
   2. Si es menor a 5.1, descarga Windows Management Framework 5.1
   3. Link: https://aka.ms/wmf5download

PROBLEMA: "Acceso denegado" o "Access Denied"
SOLUCIÓN:
   1. Asegúrate de ejecutar PowerShell como Administrador
   2. Haz clic derecho en PowerShell → "Ejecutar como administrador"

PROBLEMA: El instalador se cierra inmediatamente
SOLUCIÓN:
   1. Haz clic derecho en Instalar-Dependencias.bat
   2. Selecciona "Ejecutar como administrador"

================================================================================
  VERIFICACIÓN POST-INSTALACIÓN
================================================================================

Después de instalar, verifica que todo funcione:

1. Ejecuta el verificador del sistema:
   .\Tools\Verificar-Sistema.ps1

2. Deberías ver:
   ✓ Permisos de administrador
   ✓ PowerShell 5.1 o superior
   ✓ Módulo UniversalDashboard instalado
   ✓ Puerto 10000 disponible
   ✓ Carpetas creadas correctamente

3. Inicia el dashboard:
   .\Iniciar-Dashboard.bat

4. Abre tu navegador en:
   http://localhost:10000

5. Deberías ver la interfaz del dashboard con:
   - Nombre de la PC actual
   - Botones de configuración
   - Secciones por categoría

================================================================================
  DEPLOYMENT EN MÚLTIPLES PCs
================================================================================

OPCIÓN A: Carpeta de red compartida (RECOMENDADO)

   1. Coloca WPE-Dashboard en una carpeta compartida de red:
      \\servidor\IT\WPE-Dashboard

   2. En cada PC, crea un acceso directo que apunte a:
      \\servidor\IT\WPE-Dashboard\Iniciar-Dashboard.bat

   VENTAJA: Una sola copia del código, actualizaciones centralizadas
   DESVENTAJA: Requiere conexión a red

OPCIÓN B: Copia local en cada PC

   1. Copia la carpeta completa a cada PC:
      C:\WPE-Dashboard

   2. En cada PC, ejecuta:
      C:\WPE-Dashboard\Instalar-Dependencias.bat

   VENTAJA: Funciona sin red, más rápido
   DESVENTAJA: Debes actualizar cada copia individualmente

================================================================================
  CONCEPTO IMPORTANTE: EJECUCIÓN LOCAL
================================================================================

RECUERDA: El dashboard ejecuta scripts EN LA PC DONDE SE EJECUTA

Ejemplo correcto:
   → Quieres crear un usuario en PC "POS-MERLIOT"
   → Conéctate a "POS-MERLIOT" (RDP, físicamente, etc.)
   → Ejecuta el dashboard EN "POS-MERLIOT"
   → Crea el usuario desde la interfaz web
   → El usuario se creará en "POS-MERLIOT" ✓

Ejemplo INCORRECTO:
   → Quieres crear un usuario en PC "POS-MERLIOT"
   → Ejecutas el dashboard en PC "ADMIN-PC"
   → Creas el usuario desde la interfaz web
   → El usuario se creará en "ADMIN-PC" (no en POS-MERLIOT) ✗

REGLA DE ORO: El dashboard NO es una herramienta de gestión remota.
              Ejecútalo EN la PC que quieres configurar.

================================================================================
  CHECKLIST DE INSTALACIÓN
================================================================================

Usa este checklist para asegurarte de que todo está correcto:

□ Windows 10/11 o Windows Server 2016+
□ Permisos de administrador
□ PowerShell 5.1 o superior verificado
□ Módulo UniversalDashboard.Community instalado
□ Carpetas Logs, Backup, Temp creadas
□ Puerto 10000 disponible
□ Dashboard inicia sin errores
□ Interfaz web accesible en http://localhost:10000
□ PC actual se muestra correctamente en el dashboard
□ Logs se generan correctamente

Si todos los items tienen ✓, la instalación está completa.
